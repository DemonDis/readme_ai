Действуй как опытный системный аналитик и Senior Backend-разработчик. 

Твоя задача — спроектировать REST API для системы и представить его в виде подробной таблицы взаимодействий, а также описать схемы запросов и ответов (Request/Response payload).

Контекст и бизнес-логика проекта:
[ВСТАВЬ СЮДА ОПИСАНИЕ: Что за система? Какие компоненты общаются друг с другом? Какие основные функции нужно реализовать? Например: "Это сервис доставки еды. Мобильное приложение клиента запрашивает список ресторанов у бэкенда, а бэкенд отправляет заказ в систему ресторана"].

Пожалуйста, предоставь ответ в следующем формате:

### 1. Таблица взаимодействий API (Markdown формат)
Создай таблицу со следующими столбцами:
*   **ID** (порядковый номер)
*   **Система-Инициатор** (Кто отправляет запрос)
*   **Система-Приемник** (Кто обрабатывает запрос)
*   **HTTP Метод** (GET, POST, PUT, PATCH, DELETE)
*   **Endpoint** (URI маршрута, например `/api/v1/users/{id}`)
*   **Назначение** (Краткое описание, что делает этот вызов)
*   **Query/Path параметры** (Если есть)
*   **Статусы ответов** (Например: 200, 201, 400, 404, 500)

### 2. Схемы данных (Data Schema)
Для каждого эндпоинта из таблицы выше опиши структуру данных в формате JSON с комментариями. Включи следующее:
*   **Headers:** (Например, авторизация Bearer Token).
*   **Request Body:** JSON-схема тела запроса с указанием типов данных (string, integer, boolean, object, array) и пометкой об обязательности полей (required).
*   **Response Body (Success):** Пример успешного ответа.
*   **Response Body (Error):** Пример стандартного ответа с ошибкой.

Сделай названия полей в JSON понятными (snake_case или camelCase — выбери один стиль и придерживайся его). Добавь короткие комментарии к неочевидным полям прямо внутри JSON.
